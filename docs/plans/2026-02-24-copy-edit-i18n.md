# Copy-Edit Sprint + Internationalisation (i18n) Refactor

## Context

This plan covers two tightly linked efforts:

1. **i18n refactor** — move all user-visible copy into a centralised strings file (`src/content/strings.ts`) with English and French variants, wiring up each page to read from it.
2. **Copy-edit sprint** — a live collaborative session where Sam and Margaux review and edit every page's copy directly, watching changes update in real time via the Astro dev server.

These are sequenced: the i18n refactor must land first so the copy-edit sprint has a single file to work in.

---

## Phase 1: i18n Refactor

### Goal

Every piece of user-visible text on every page is sourced from `src/content/strings.ts`. Nothing is hardcoded in `.astro` files. The strings file is the single source of truth for copy in both languages.

### File structure

```
src/
  content/
    strings.ts        ← all copy, EN + FR, keyed by page/component
  lib/
    i18n.ts           ← thin helper: t(key, lang) → string
```

### Language selection

The active language is determined by `Astro.locals.lang` (or a fallback to `'en'`). The `global.i18n` feature flag controls whether French is accessible. When off, `lang` is always `'en'`.

Language can be set via:
- A `?lang=fr` query param (stored in a cookie for persistence)
- Or a toggle in the nav (post-design-sprint)

This is consistent with the existing `global.i18n` feature flag.

### Steps

1. Audit all `.astro` pages — extract every hardcoded string into a spreadsheet/list
2. Create `src/content/strings.ts` with all EN values (FR can be empty strings initially)
3. Create `src/lib/i18n.ts` helper
4. Update each page to use `t()` instead of hardcoded copy
5. Add French translations (can be done page-by-page or all at once)
6. Wire `lang` into `Astro.locals` via middleware
7. Tests: check that all pages render without missing keys; existing accessibility tests should still pass

## Phase 2: Copy-Edit Sprint

### Goal

Sam and Margaux review and refine every page's copy together, seeing changes live.

### Workflow

1. Run `npm run dev` (http://localhost:1213)
2. Open two windows side by side:
   - Left: browser at localhost:1213
   - Right: `src/content/strings.ts` in the editor
3. Work through pages one at a time — edit a string in `strings.ts`, save, watch the page update instantly (Astro HMR)
4. For French: toggle `?lang=fr` in the browser to preview French copy

### Pages to review (in order)

1. Home / placeholder page
2. NYC overview
3. NYC RSVP form
4. NYC Schedule
5. NYC Travel
6. France overview
7. France RSVP form
8. France Travel
9. Registry

### What to check on each page

- Tone: composed, direct, architectural (per brand identity doc)
- No exclamation marks, no "to follow", no ceremony language
- Date formats consistent (NYC: `October 11, 2026` / France: `May 28–30, 2027`)
- French copy reviewed by Margaux
- Error messages and form labels reviewed
- Email confirmation copy reviewed (in `src/lib/email-templates.ts`)
